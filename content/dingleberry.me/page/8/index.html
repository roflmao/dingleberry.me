free internet on cafes or airports with dnstunnel or icmp ping tunnel

Posted November 3, 2008 by MrDingle

aptitude install libmime-base32-perl libwww-perl libnet-dns-perl
to use this server as a http proxy (which is what we will do here), do the following as well:
aptitude install privoxy
wget http://www.doxpara.com/ozymandns_src_0.1.tgz
tar xvzf ozymandns_src_0.1.tgz
./nomde.pl -i 0.0.0.0 hesten.XXXXXXXX.no
on client, follow the same procedure to install packages and download scripts etc…

to get a shell on the dns server:
ssh -C -o ProxyCommand="./droute.pl -v sshdns.hesten.XXXXXXXX.no" user@localhost
to use the proxy we installed on the dns server:
ssh -C -o ProxyCommand="./droute.pl -v sshdns.hesten.XXXXXXXX.no" -NL 8118:localhost:8118 user@localhost
open a new window and :
export http_proxy=http://localhost:8118
and launch your fav browser..

For a full, and more thorough guide – check dnstunnel.de

Setup smarthost with postfix

Posted October 16, 2008 by MrBerry

Okey, so you have a server behind a ISP that blocks outgoing connections to port 25, or you simply want to route all mail through another smtp. What do you do? Smarthost is the answer.

This can easily be done with postfix:

apt-get install postfix libsasl2-modules mailx

Edit postfix config: /etc/postfix/main.cf
#here we use mail.whatever.com port 587, change as needed
relayhost = [mail.whatever.com]:587
#the next lines are only needed if you want to use tls and/or smtp auth on your relayhost
smtp_use_tls = yes
smtp_sasl_auth_enable = yes
smtp_sasl_password_maps = hash:/etc/postfix/smtp_auth
smtp_sasl_security_options = noanonymous

Further for configuring smtp-auth, edit: /etc/postfix/smtp_auth
mail.whatever.com username:your_secret_password

Then you convert this to postfix hash table format and restart postfix:
cd /etc/postfix
postmap smtp_auth
/etc/init.d/postfix restart

Now to test the relay and tail your log file to see the connection go through:
echo "no soup for you!" | mail your@mailadress.com
tail -f /var/log/mail.log

You can read more on smarthost @ Wikipedia
And we are done here!

Setting up mysql replication

Posted October 15, 2008 by morten

This is actually pretty easy. It is well documented here, but I’ve made a quicklist :

Edit my.cnf on the master. It should contain :
server-id = 1
log-bin=mysql-bin

Edit the my.cnf on the slave. It should contain :
server-id = 2

First grant access to a replication user on the master server:
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'slavehost.example.com' IDENTIFIED BY 'password';

Lock the master:
FLUSH TABLES WITH READ LOCK;

Get the master status for later use :
SHOW MASTER STATUS;

Dump the master db with mysqldump:
$ mysqldump mybase > mybase_out.sql

Import the master base on the slave :
$ mysql mybase < mybase_out.sql

Set the master info on the slave :
CHANGE MASTER TO MASTER_HOST='masterhost.example.com', MASTER_USER='repl', MASTER_PASSWORD='password', MASTER_LOG_FILE='mysql-bin.000001', MASTER_LOG_POS=13963;

Start up the slave :
START SLAVE;

And finally release the master lock by exiting your mysql session.

Annoyed by the output of “locate”

Posted October 15, 2008 by morten

I like to have colors and additional pr-file-info when searching files when not having the time to wait for “find” to complete.
I made a silly little sh script, named it “clocate” and put it in my /usr/local/bin

#!/bin/sh
if [ -n $1 ]; then
for fil in `locate $1`; do
ls --color=auto -l $fil
done
fi

It does the job for me.
